# F99-6 AI Refiner

This phase defines a finite, fail-closed contract for a post-vectorization refiner that may correct only bounded, allowlisted defects and must never reduce fidelity.

## Qualified evidence

The contract requires at least 24 uniquely identified cases. Every case records immutable input and model revisions, pre- and post-refinement fidelity, edit count, deterministic output digest and defect counters.

## Acceptance thresholds

- Mean fidelity gain must be non-negative.
- No individual case may lose fidelity.
- p05 post-refinement fidelity must be at least `0.990`.
- Edit count must not exceed `64` per case.
- Only `geometry`, `color`, `topology` and `artifact_cleanup` edits are allowed.
- Topology damage, provenance gaps, output digest drift and source artifact mutation counts must be zero.

Missing identifiers, duplicate cases, invalid SHA-256 revisions, non-finite or out-of-range metrics, unbounded edits, unexpected edit classes, corpus shrinkage or any regression fails closed.
